# Security Policy

## Reporting a Vulnerability

The security of Azure DevOps PR Viewer is important to us. If you discover a security vulnerability, please help us by reporting it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by:

1. Opening a new issue on GitHub with the title "Security: [Brief Description]"
2. In the description, provide as much information as possible:
   - Type of vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)

Alternatively, you can email the maintainer directly through the contact information on the [GitHub profile](https://github.com/johncwaters).

### What to Expect

- We will acknowledge your report within 48 hours
- We will provide an estimated timeline for a fix
- We will keep you informed of our progress
- We will credit you in the release notes (unless you prefer to remain anonymous)

### Supported Versions

We release security updates for the latest version of the extension. Please ensure you're running the most recent version before reporting a vulnerability.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Security Best Practices

When using Azure DevOps PR Viewer:

- Keep the extension updated to the latest version
- Never share your Azure DevOps personal access tokens
- Use the extension's built-in Microsoft Entra ID authentication
- Review the extension's permissions in VS Code
- Report any suspicious behavior immediately

Thank you for helping keep Azure DevOps PR Viewer and its users safe!
